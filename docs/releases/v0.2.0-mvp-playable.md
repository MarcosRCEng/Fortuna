# v0.2.0 - Playable MVP

## Summary

`v0.2.0 - Playable MVP` consolidates sprints 14 through 23 into the first
end-to-end playable Fortuna experience. The milestone connects simulated market
data, wallet and portfolio operations, missions, Mentor Fortuna, Cidade Fortuna,
demo seed data, API coverage, and release/compliance documentation.

## Scope

- Persistence with Prisma/PostgreSQL, selectable by environment.
- MVP financial API and Bruno request collection.
- Web UI for wallet, market, orders, missions, history, and city progression.
- Game loop services for progression, events, income, unlocks, and city state.
- Educational missions and mentor feedback.
- Mock market provider with future-ready provider, cache, and fallback
  architecture.
- Structured logs, correlation id support, readiness checklists, and demo data.

## Financial Invariants

The release keeps the core financial model intentionally conservative:

- All money is represented as integer cents.
- Floats are not a source of truth.
- Negative balances are blocked.
- Purchases without sufficient balance are blocked.
- Sales above the current position are blocked.
- Leverage is not supported.
- The UX and API include educational framing and simulated data.

## Persistence And Providers

The API can run with in-memory persistence for fast local testing or Prisma for
PostgreSQL-backed demo state. Market data remains mock-only by default. External
provider paths are placeholders for future compliance-reviewed integrations and
are not enabled as real investment integrations in this release.

## Validation Commands

Required validation for the integration PR:

```bash
corepack pnpm install
corepack pnpm build
corepack pnpm test
corepack pnpm test:coverage
corepack pnpm test:e2e
corepack pnpm prisma:generate
```

Optional PostgreSQL validation:

```bash
docker compose up -d
Copy-Item .env.example .env
corepack pnpm prisma:migrate
corepack pnpm db:seed
```

For Prisma-backed API runs, set:

```bash
FORTUNA_PERSISTENCE=prisma
```

## PR Readiness Gate

The PR should leave draft only after:

- Build passes.
- Tests pass.
- Release documentation exists.
- `docs/release-checklist.md` has been reviewed for the final validation run.

## Non-Goals

- No brokerage, custody, order routing, investment account, or real money
  integration.
- No personalized financial recommendation.
- No live market dependency required for the MVP flow.
